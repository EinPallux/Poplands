# Changelog

All notable changes to **Poplands** are documented here.
Format: [Keep a Changelog](https://keepachangelog.com/en/1.1.0/) · Versioning: milestone-based `0.x` (see [ROADMAP.md](ROADMAP.md)), aiming for semantic versioning from 1.0.

## [Unreleased]

### Added — v0.6 "Living Canvas" (in progress)
- **River/path auto-tiling (S10):** path and river tiles now **connect** — lay a run and it resolves into straight/corner/tee/cross/end pieces instead of isolated squares. A pure `resolveTileShape(mask)` (the standard 16-mask blob-autotile reduction, unit-tested for all 16 neighbour masks) + per-kit GLB variant data (`content/tileKits.ts`); `PropRenderer` reads a tile's 4 same-kit neighbours, swaps the instanced pool's variant, and ripples the update to neighbours on place/remove/move. Uses the existing Nature Assets 2 connectivity kit (12 new tiny tiles, CC0). **Placements stay the truth — no sim/save change.** Shape resolution verified end-to-end (a tee junction, three ends, straight channels all resolve correctly); the GLB base-orientation calibration is a real-hardware eyeball item. (`stone-path` has only corner/end art, so it degrades gracefully to its plain tile for other shapes.)
- **Edge-anchor placement (S8):** docks, jetties, and moored boats now **must straddle the island's edge** — their footprint needs ≥1 on-island anchor cell and ≥1 off-island cell, so they hang out over the cloud sea. A new `ItemDef.edgeAnchor` flag + a pure `edgeAnchorOrigin(anchor, fp, rot)` grid helper that shifts the origin back for -X/-Z rotations, so a dock reaches **all four** coastlines (not just north/east) as you rotate it. `canPlace` gains a `needs-edge` reason; `BuildSession` routes hovers/clicks through the origin shift. Occupancy is tracked on the off-island overhang too, so two docks can't share it — and it stays claimed even if you later buy the chunk underneath. `decor.dock`/`decor.jetty`/`decor.moored-sloop` are edge-anchored.
- **Tiers 11–14 content (62 items):** the mid/late catalog fills out across four themed sets — **Harbor/Coastal** (Tier 11: leaning/coconut palms, coastal boulders, rowboat, jetty, raft, treasure chest, cannon, boathouse, beached shipwreck…), **Spooky** (Tier 12: five headstone shapes, obelisk, coffin, jack-o'-lantern, brazier, iron fence/gate, dead pines, mausoleum, keeper's crypt…), **Winter/Snowcap** (Tier 13: snow pines, spruce, ice rocks, frozen pond, firewood, camp tent, log cabin, alpine chalet, warm lodge, cocoa stall…), and **Grand Town** (Tier 14: clock tower, castle, temple, grand fountain, monument, town hall, grand market, grain mill, guildhall, manor, townhouse…). `ItemDef.tier` widened to 14 (Tier N unlocks at Level N; gating + "New!" reveals extend automatically). All 62 shipped via the manifest pipeline (Kenney/Quaternius CC0), **every scale computed from the pipeline AABBs**. **Research fanned out across four parallel content-scout agents** that inspected the raw packs and returned verified paths + sizes.
- **Settings & accessibility (S23):** a **frame-rate cap** (Off / 30 / 60 fps) in the loop for battery/thermals, plus the **Time of day** freeze from v0.5 — both persisted. Reduced-motion continues to gate every juice preset + ambient event. (UI-scale and colour-blind cue passes are still open.)
- **Island Album (S21):** a cozy scrapbook (📖 button / **J**) that surfaces the lifetime **milestone counters** with tier progress, the **Neighbours** who've moved in, the **Pals** who visit, and a **biome tally** of your districts. Read-only, pulls a fresh snapshot on open, all text via the string table.
- **Photo mode (S23):** press **P** (or the button) to **hide all UI**, drop a soft "diorama" **vignette**, and **export the current frame as a PNG**. Esc exits. The capture renders one fresh frame and reads the canvas back, so it works without a preserved drawing buffer — the marketing shot takes itself.
- **Theme-driven touches (S19/S20/S7):** three biome payoffs. **Theme-specific secrets** — the per-chunk secret roll now reshuffles *which kind* wins by biome (Sandbar favours beachcombing digs, Spooky haunted chests, Snowcap rare flora) while every table holds the "nothing" slice at exactly 50%, so the island-wide secret *rate* and no-grind economy are provably unchanged (new distribution tests pin it). **Theme ambient emitters** (`ThemeAmbience`) — a Spooky Grove gets faint ground mist + nocturnal bats, a Snowcap gets gentle **snowfall** (one `Points` draw), a Sandbar gets sun-glinting sand drift; all fixed-size pools (cost independent of island size), driven by `nightFactor`, reduced-motion-safe, self-wiring to `chunk:unlocked`. **Themed slab rock** — the skirt's rock bands now re-tint per biome (icy blue Snowcap, sandy Sandbar, desaturated Spooky) via the same per-outline-point sampling as the grass lip; meadow stays pixel-identical.
- **Themed chunks (S7):** grown districts now come in four biomes — **Meadow · Sandbar · Spooky · Snowcap** — each with its own lawn + grass-lip palette. A deterministic `themeFor(seed, chunk)` rolls the biome (≈58% meadow, the rest sprinkled) so the world is reproducible and a **Survey chip previews the biome** (🌱🏖️🌙❄️) before you buy. Theme threads cleanly: stored per-chunk in `IslandModel` + the save, `GroundBuilder` tints each chunk's lawn, and `SlabBuilder` re-tints the grass lip **per outline point** so biome boundaries read crisply. `chunk:offered`/`chunk:unlocked` carry the theme. (Theme-specific secrets & ambient emitters, and themed slab *rock*, are follow-ups.)
- **v0.6 still open (honest scope):** river/path **auto-tiling**, the **edge-anchor** placement rule (docks/moored boats), **themed lazy-loading** (S4), the **Tiers 11–14 content** push (~60 items), and the UI-scale / colour-blind accessibility passes are **not yet done** — v0.6 has its headline breadth (biomes, album, photo mode) but is not feature-complete.

### Added — v0.5 "Little Neighbours" (feature-complete — awaiting milestone review)
- **Ambient life (S19/S20):** the diorama breathes on its own now — **fireflies** twinkle low over the island at night, the odd **shooting star** streaks the dark sky (a soft wish chime + sky sparkle, `event:shootingStar`), and a stray **hot-air balloon** drifts past by day. All driven by the day-night `nightFactor` on long random timers (never busy), reduced-motion-safe.
- **Ambient audio (S22):** a single, well-spaced **birdsong chirp** by day / **cricket trill** by night — the "zero machine-gun artifacts" DoD by construction (one rate-limited voice, 4–10 s apart). Plus a happy **pet chirp** and the tap-to-greet **babble**.
- **Liveliness dividend (S13):** a lively island quietly pays a little extra — every 10 s each resident Islander & Pal chips in a couple of Pops (capped at 18 souls, time-gated so it's never farmable). `LivelinessSystem` is three.js-free + unit-tested.
- **Islanders (S16) — the milestone identity:** little neighbours now **move in as your island gains homes** and **wander it** while you build. Population = **min(Σ house capacity, 12)** and is **monotonic** — removing a home never evicts anyone (no-fail-state covenant). A 16-name roster (`src/content/roster.ts`, move-in order Mo → Pia → Bram → Lulu → …) maps onto the 12 lit, skinned Pack-2 characters; names route through the string table (i18n-ready).
- **Sim/render seam held (Tier C):** `IslanderSystem` (sim, **three.js-free**) owns pure kinematics — plain `{x,z}` positions, headings, per-agent decision timers — and a straight-line "stroll to a nearby walkable cell, re-decide on arrival/obstruction" wander (no A* needed on a small open grid; decisions are seconds apart, only the position integration is per-frame). It exposes an `agents` snapshot the renderer **pulls** each frame (like WorldFx reads economy). New `IslandModel.walkable(wx,wz)` — on-island, not landmark-blocked, free of a solid prop (paths are strollable) — keeps Islanders off benches, huts, and the void.
- **`AgentRenderer` (world):** projects the snapshot into animated meshes — one `AnimationMixer` per agent, an eased **idle↔walk crossfade** driven by `moving`, and **anti-sync** (±10% mixer timescale + random clip phase) so a crowd of the same model never marches in lockstep. Each model is **auto-seated on the ground** via its AABB (no per-model offset guessing). Clones share geometry/materials with the cache.
- **`AssetRegistry` now keeps animation clips** (previously dropped on load) and adds a **skinned-safe clone path** (`cloneAnimated` via `SkeletonUtils.clone`) so each agent's skeleton rebinds independently — a plain `scene.clone()` would share one skeleton and deform every instance in lockstep. Pipeline gained a **required-clip gate** (`npc.*`/`pal.*` must ship `idle`+`walk`) so a bad asset swap fails loudly, not as a silent T-pose.
- **Save v4:** the Islander roster slice (`{ residents }`) + a v3→v4 migration (the sim re-derives residents from home capacity on first start); `normalize` deep-defaults it. Live positions aren't persisted — Islanders simply re-scatter on load.
- **Move-in welcome:** a gentle sparkle + chime where a neighbour appears (`npc:arrived`); the full walk-in-from-the-edge move-in *moment* lands in a later v0.5 slice.
- **Tap-to-greet (user feedback):** click an Islander and they **pause, say a cute one-liner in a springy speech bubble, play a wave/nod emote, and babble** a little WebAudio blip. 14 cozy lines (`content/chatter.ts`, string-table + i18n-ready), a random emote from `emote-yes`/`interact-*`, a one-shot clip layered over the idle/walk blend, and a world-anchored `SpeechLayer` bubble that tracks its speaker. Picking is a raycast against agent meshes, gated so it never fires mid-build. `cmd:clickNpc` → `npc:spoke`/`agent:playClip`.
- **Fixed (user feedback):** Islanders **walked backwards** (facing forward, moving the other way) — the model-yaw offset was a half-turn off; corrected so they face their travel direction.
- **Move-in moment:** a new resident now **enters from the island's cloud edge and strolls in toward the heart of the island**, with a named welcome toast ("Mo moved in! 🏡") + a sparkle/coin burst. Only genuine move-ins do this (load re-spawns scatter quietly).
- **Day-night cycle (S7):** a slow **dawn → day → dusk → night** loop (4-minute day) keyframes the two lights, the sky dome gradient, the cloud sea + puff clouds, and the fog — the whole frame moves together, not just the island. The sun's *direction* stays fixed (no per-frame shadow refit — S5); only colour, intensity, and the gradient move, which carries the mood cheaply. A **Time of day** setting freezes it (Cycle / Day / Dusk / Night), persisted. `TimeOfDay` publishes a `nightFactor` for glow + night-only events.
- **Lantern glow (S7/S20):** soft additive halos fade in over lanterns, lampposts, campfires, and the lighthouse as dusk falls — the "windows glow at dusk" beat. `GlowLayer` tracks placements via the bus (new lamps light up), one shared radial texture, opacity driven by `nightFactor`. **The night beauty shot lands** (ART §3.2): moonlit island, deep-indigo sky, dark cloud silhouettes, warm lamp pools.
- **Pals (S18):** six animals — cat, dog, bunny, chick, pig, cow — **scamper onto the island as it gets lively** (one Pal drawn per 8 nature items, capped at 6, monotonic). `PalSystem` (sim, three.js-free) mirrors the Islander wander with a quicker gait, the occasional grass **nibble** (`eat` clip), and **petting**: tap a Pal → it plays a happy `gesture-positive`, puffs pink **hearts**, and chirps. Rendered by a second `AgentRenderer` at a smaller target height (the same Tier-C pipeline, now per-model scale-normalized so characters and pets share one renderer). New pet assets shipped via the pipeline (Cube Pets CC0). Save slice extended with a monotonic `pals` roster; events `pal:adopted`/`cmd:clickPal`/`pal:petted`.
- **Verification:** `scripts/verify-v05.mts` — boot → no neighbours without homes → build 3 huts → 3 Islanders move in (roster + a mesh each, Mo first) → they wander (Δ>0.4 blocks) staying on walkable ground → roster persists across reload + agents re-spawn (10 checks, zero page errors under swiftshader — skinned clone + mixer confirmed). 9 `IslanderSystem`/`walkable` unit tests (population, Σ-houses capacity, monotonic no-evict, 12-cap, re-spawn-no-dup, a 3000-step on-island wander soak). **121 tests green.**
- **Next v0.5 slices:** the move-in *moment* (walk in from the cloud edge + popup), Pals + petting, the day/night cycle, the full ambient-audio pass, and ambient events (shooting stars). Islander facing offset (`MODEL_YAW_OFFSET`) is a one-constant cosmetic — worth an eyeball on real hardware.

### Added — v0.4 "New Horizons" (in progress — core loop complete)
- **Chunk expansion (S7/S8):** the island grows. `IslandModel.addChunk`/`expandableSlots` grow the lattice; the base slab + wobbly outline **re-trace organically** for any chunk shape (no code change — they already derived from the block set). `ExpansionSystem` offers up to 3 **Surveys** on the free edges (seeded, re-rollable for a token Pop fee), validates + charges both wallets + grows + refreshes offers. Pricing per CONTENT §6.2 — Pops `round10(250·1.32^(k−1))` (then ×1.13 from k=9), Stardust `2+⌊k/4⌋`, 36-chunk soft cap. Persists to `save.island.chunks`.
- **Chunk-arrival set piece (S11/S12, ART §7.2):** a bought chunk is built as a standalone slab and **rises out of the cloud sea, docks with a spring bounce** (`dampedStep` ζ≈0.55), then confetti + fanfare + a sparkle grass-ripple. ~2.2 s, soft input-lock for the first 0.8 s (`juice:setPiece*`) — the camera keeps easing (never shakes). On dock the merged island is rebuilt and the temp slab disposed. Reduced-motion collapses to particles-lite inside the preset. `island:grew` eases the camera out (`frameIsland`), refits the shadow box, and recenters the sky.
- **Secrets & discoveries (S19):** a deterministic per-`(seed,chunk)` roll (dig 35% · chest 10% · flora 5% · nothing 50%, GDD §10). A dig takes 3 escalating clicks, a chest/flora one; on open, rewards flow through the bus (● ✦ credit, `discovery` XP, the `findSecret` quest + `secretsFound` milestone). Starter chunks and the first bought chunk seed a guaranteed friendly dig. World-anchored ✨ markers; nothing expires; found latches.
- **Save v3:** `SaveSecret` slice + `secretsFound` counter + a v2→v3 migration (the sim does the content roll on first start); `normalize` deep-defaults the new fields.
- **Survey/secret UI (S21):** world-anchored "call this chunk" balloon chips (live-priced) and ✨ secret markers, projected each frame; `pc.treasure` postcard exercises `findSecret`.
- **Verification:** `scripts/verify-v04.mts` — boot → surveys priced 250●/2✦ → buy → grows to 5 chunks → dig the (0,0) secret → +100 ● → `secretsFound=1` → persists across reload (12 checks). Existing tut.* ids preserved (save-safe); the capstone re-links forward.

### Added — v0.3 "Pops & Purpose" (pending user milestone review)
- **Economy (S13):** two currencies (Pops ●, Stardust ✦) with a single-writer wallet. Income buildings accrue Pops lazily — `computeStored` is a pure read over elapsed time clamped to each building's cap, so offline is never special-cased and a backward clock costs nothing (no-grind covenant). Collect one (`cmd:collect`) or all ripe (`cmd:collectAll`); placement charges & 100 %-refunds through one path. Purchases are gated, never blocked mid-drag. 12 economy tests (accrual, cap clamp, offline, refund symmetry).
- **Progression (S14):** XP + levels to 20 on a `round10(60·L^1.55)` curve; placement grants XP once per placement id (idempotence ledger, so move/hydrate never double-count); quest & level rewards feed the same intake. Tier gating — Tiers 1–2 open at L1 (the tutorial teaches them), Tier N≥3 unlocks at level N — with a live "New in Catalog" reveal. 13 progression tests.
- **Quests (S15):** data-driven predicate DSL (`place/own/collectPops/reachLevel/adjacency/findSecret/all`) evaluated by a three.js-free runtime. 12-step tutorial chain (auto-complete, no claim button), a rolling 2-slot postcard pool with cooldown refill + skip-defer, and lifetime milestone counters. Forgiving seeding counts already-owned items; the move exploit is closed (a re-drop never advances counters). 11 quest tests.
- **Content Tiers 3–6 (~40 items):** crops, homes, décor, and income buildings as data, plus a **prefab bake pipeline** composing multi-part landmarks (Bakery, Well, Scarecrow, Apple Tree) from kit pieces into single GLBs.
- **Full HUD & juice (S21/S11):** wallet pills with spring-tick counters, an SVG level ring bound to XP progress, the Mailbox (tutorial + postcard cards, skip), world-anchored ripen bubbles that fill as income accrues, coin-arc collection flight, level-up & quest-complete celebrations. All motion via presets with the reduced-motion path.
- **Verification:** `scripts/verify-v03.mts` — headless boot → tutorial-advances-on-place → charges+rewards → Tier-2 income building buildable at L1 → wallet math → reload persistence (12 checks green). `scripts/shots-v03.mts` captures milestone beauty shots.

### Changed — feel polish (user feedback, 2026-07-11)
- **Bouncier placements:** the `popBounce` pop-in easing now overshoots to ~1.19 (from ~1.16) with a slightly longer-lived settle dip, over a 0.36 s pop — a clearer "boing" on every placement without tipping into wobble.
- **Build-bar tab transitions:** switching category tabs now replays a staggered card cascade (rise + scale + fade, springy easing, 24 ms per-card delay, reduced-motion safe) so the catalog feels alive instead of a static show/hide.
- **Review hardening (first pass):** postcard slots refill once their cooldown lapses (a per-frame quest tick), the tutorial funds its own XP so reaching L2/L3 is never a grind wall, a skipped postcard is deprioritised behind the fresh pool, and the XP idempotence ledger is bounded to live placements so it can't grow without limit.
- **Review hardening (second pass):** a fresh 3-agent adversarial review (economy · save/HUD · progression-quests) with independent per-finding verification landed four confirmed fixes — (1) `collect()` no longer regresses the accrual anchor on a backward wall clock (`collectAll` could otherwise over-credit up to a building's cap on clock recovery); (2) `normalizeV2` deep-defaults present-but-partial save slices so a hand-edited/imported save can't brick boot; (3) the per-frame quest tick is gated behind a `needsRefill` flag so it never re-filters the postcard pool every frame (both the WorldFx bubble pass and the quest tick are now steady-state allocation-free per TECH §6.5); (4) `QuestSystem.level` advances monotonically so a nested multi-level reward can't regress it. 3 new regression tests (92 total).

### Decided (v0.3 review, 2026-07-11)
- **Place→remove self-grind is accepted, not a bug.** The review surfaced that 100% refunds + never-revoked placement XP/milestone counters let a player grind XP/levels/Stardust via place→remove churn. Deliberately left unpoliced (cozy single-player, self-inflicted, genre-normal); documented in GDD §7.5 and SYSTEMS §S14. Revisit only if the v0.4 chunk-price Stardust sink shows it distorts expansion pacing in playtests.
