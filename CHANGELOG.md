### Added ‚Äî Fishing pond (post-1.0)
- **A gentle cast-and-catch minigame.** Place a **Fishing Pond** (Tier 5, `nature.fishing-pond`) and tap it to cast a line; after a short wait a fish nibbles (the bobber jiggles, a "!" pops), and tapping again within a generous window reels it in. Miss the window and the fish just "gets away" ‚Äî no penalty, recast freely (the no-fail covenant, GDD ¬ß7.5). A new three.js-free **`FishingSystem`** (sim) owns the cast‚Üínibble‚Üícatch state machine with an injected RNG (deterministic for tests), a weighted **11-species catch table** (`content/fish.ts` ‚Äî commons up to a legendary **Star Sturgeon**, plus a cheeky **Old Boot**), and the catch collection. Rewards (Pops + Stardust on the rarer fish + XP) flow through the same reward-event pattern as quests/secrets ‚Äî `fishing:caught` is credited by Economy + Progression ‚Äî so the sim stays decoupled.
- **Fish Journal (üé£ button / `F`).** A cozy scrapbook of every species: caught ones show icon + name + rarity + count, the rest tease as locked "‚ùì" silhouettes, under a "X / N species ¬∑ Y catches" header. Mirrors the Island Album (read-only, fresh snapshot on open).
- **Juice.** A world-anchored **bobber** that bobs on the water and jiggles on a bite, a "Tap to reel it in!" prompt, a rarity-tinted **catch popup** (big icon + name + reward, "New!" badge on a first catch) that pops and rises, a water sparkle + dust-ring splash on the cast, and three new SFX (cast splash / nibble blip / reel flourish). Reduced-motion holds the bobber still and drops the popup to a static readout.
- **Save v5.** A new `fishing` collection slice (species ‚Üí count + lifetime total) with a v4‚Üív5 migration + normalize default, so older saves back-fill to an empty log (no data loss). Persists on every catch.
- **Content + wiring.** The pond is one new manifest model (a re-skinned water tile from the CC0 Nature pack, `early` wave so boot stays 2.36 MB); play-mode clicks route through `BuildSession` (a pond ‚Üí `cmd:castLine`, else the existing income-collect). **Verified** by `scripts/verify-fishing.mts` (10 checks: pond loads + places, click‚Üícast routing, nibble, catch +1, popup, journal) + 8 `FishingSystem`/roll unit tests + 2 save tests. **162 tests green**; check + build clean; all prior verifies still pass.

### Added ‚Äî Cozy polish batch (post-v0.7, in progress)
- **Weather ‚Äî passing showers + a rainbow (post-1.0 backlog).** The sky occasionally opens into a gentle shower and clears to a low-poly **rainbow** arching over the diorama. A new `WeatherSystem` (world layer): rain is one `LineSegments` draw of ~320 streaks over the current island bounds that eases in, lingers ~15‚Äì25 s, then fades; when a daytime shower clears it leaves a **vertex-coloured half-annulus rainbow** (red‚Üíviolet across the band) that **billboard-yaws to the camera** so it reads from any orbit angle and lingers ~11 s. Rare and slow by construction (a shower every 1‚Äì3 min, never busy), self-wires to `chunk:unlocked` so the curtain always covers growth and the arch stays centred, zero per-frame allocation, and **reduced-motion holds the sky calm** (no rain, no fades). Adds ‚â§2 draws only while a shower is on screen. Verified in `scripts/verify-cozy.mts` (rain + rainbow both draw, no errors) with a rain-and-rainbow beauty shot.
- **Prominent income bubbles ‚Äî "how much is collectible" at a glance.** The ripen bubble over an income building now surfaces a **readable cream pill showing `‚óè amount`** ‚Äî but only once the building is *worth a look* (‚â•55% ripe **and** ‚â•12 Pops banked), so a busy island stays calm. Below that threshold it's the same subtle fill-dot as before; a fully-ripe prominent pill gently pulses/glows to invite collection. `WorldFx` keeps deriving bubbles from economy state each frame (so it's correct on load / after moves), with the amount text cached by rounded value to avoid per-frame DOM churn. Threshold-gating verified headlessly (full building ‚Üí pill with the amount; a barely-filled one stays a quiet dot).
- **Background music ‚Äî drop in your own soundtrack.** Drop a **`public/bgm.mp3`** and it loops automatically (starting on the first user gesture, per browser autoplay policy). Entirely optional ‚Äî with no file present the new `MusicSystem` is a silent no-op and never errors. A **Music** volume slider in Settings (separate from the **Sound** SFX slider) tunes it live and **persists in the save** (`SaveSettings.musicVolume`, back-filled on older saves with no version bump ‚Äî round-trip tested). Documented in the README.

### Added ‚Äî v0.6 "Living Canvas" (feature-complete ‚Äî awaiting milestone review)
- **Themed / phased lazy-loading (S4):** assets now stream in **waves** instead of one boot download. A pure `content/assetPhases.ts` classifies every model ‚Äî **boot** (Tiers 1‚Äì4 + Islander/Pal agents + world fixtures + auto-tile variants), **early** (Tiers 5‚Äì14 with no biome), or **themed:&lt;biome&gt;** (the Sandbar/Spooky/Snowcap Tier 11‚Äì13 sets, tagged via a new `ItemDef.theme`) ‚Äî and the pipeline stamps `phase` into `manifest.json` and **hard-fails if the boot wave tops 3 MB**. First paint now waits on **2.36 MB** instead of loading all **8.58 MB** up front. `AssetRegistry` gained `loadPhase`/`ensure`/`phaseOf` + a pending-promise cache (de-dupes concurrent loads ‚Äî closes the SYSTEMS S4 double-load-race gap). Later waves trigger in the background: **idle-after-boot ‚Üí early**, **level-up ‚Üí the unlocked biome's set** (+ an early safety net), **chunk arrival ‚Üí the arriving biome's set**. Returning saves **pre-await exactly the phases their placements use** before the world is projected, so every synchronous `show()`/`promote()` keeps its throw-free contract; a still-loading model defers to a pending-show queue (and `makeGhost` falls back to a footprint box) so interactive play is never a crash. A meadow-only island therefore **never downloads** the biome sets. Verified headlessly (`verify-lazyload.mts`, 13 checks incl. a returning early-placement save + adjacent river tiles resolving to variant GLBs, all error-free).
- **UI scale + colour-blind cues (S23, GDD ¬ß11):** a **UI size** setting (Small / Normal / Large / Extra large) drives one `--ui-scale` CSS variable; each chrome widget scales around its own pinned anchor edge, so nothing drifts off-screen and the world-anchored layers stay pixel-locked.
