## [Unreleased]

### Added — Seasons (post-1.0)
- **The whole island shifts through Spring → Summer → Autumn → Winter.** A new **`SeasonSystem`** multiplies a per-season tint onto the day-night colors every frame *after* `TimeOfDay` — which re-`.copy()`s the base light/sky/fog colors each frame, so the season **modulates** the cycle without ever accumulating. Tinting the **light** (not just the lawn) shifts grass, trees, and buildings together — warm amber for autumn, cool blue for winter, fresh for spring, bright gold for summer — and layers cleanly on top of the existing biome themes. A **`SeasonAmbience`** layer adds the signature falling touch: **spring petals, autumn leaves, winter snow, and summer's lazy gold motes** — one fixed Points pool over the island bounds (cost independent of island size), re-coloured on season change, reduced-motion-safe (hidden), self-wiring to island growth.
- **Season picker in Settings** — "Auto (by month)" follows the real-world month, or pin Spring/Summer/Autumn/Winter. Persisted in `SaveSettings.season`, back-filled on older saves with **no save-version bump**. `content/seasons.ts` holds the per-season data (tint multipliers + ambient kind + emoji) as content-as-data; `resolveSeason`/`seasonForMonth` are pure + unit-tested.
- **Verification.** `scripts/verify-seasons.mts` (every season resolves + tints the light distinctly — warm autumn vs cool winter — with no page errors) plus a beauty shot per season and 4 new season unit tests. **166 tests green**; check + build clean; all prior verifies pass.

### Added — Fishing pond (post-1.0)
- **A gentle cast-and-catch minigame.** Place a **Fishing Pond** (Tier 5, `nature.fishing-pond`) and tap it to cast a line; after a short wait a fish nibbles (the bobber jiggles, a "!" pops), and tapping again within a generous window reels it in. Miss the window and the fish just "gets away" — no penalty, recast freely (the no-fail covenant, GDD §7.5). A new three.js-free **`FishingSystem`** (sim) owns the cast→nibble→catch state machine with an injected RNG (deterministic for tests), a weighted **11-species catch table** (`content/fish.ts` — commons up to a legendary **Star Sturgeon**, plus a cheeky **Old Boot**), and the catch collection. Rewards (Pops + Stardust on the rarer fish + XP) flow through the same reward-event pattern as quests/secrets — `fishing:caught` is credited by Economy + Progression — so the sim stays decoupled.
- **Fish Journal (🎣 button / `F`).** A cozy scrapbook of every species: caught ones show icon + name + rarity + count, the rest tease as locked "❓" silhouettes, under a "X / N species · Y catches" header. Mirrors the Island Album (read-only, fresh snapshot on open).
- **Juice.** A world-anchored **bobber** that bobs on the water and jiggles on a bite, a "Tap to reel it in!" prompt, a rarity-tinted **catch popup** (big icon + name + reward, "New!" badge on a first catch) that pops and rises, a water sparkle + dust-ring splash on the cast, and three new SFX (cast splash / nibble blip / reel flourish). Reduced-motion holds the bobber still and drops the popup to a static readout.
- **Save v5.** A new `fishing` collection slice (species → count + lifetime total) with a v4→v5 migration + normalize default, so older saves back-fill to an empty log (no data loss). Persists on every catch.
- **Content + wiring.** The pond is one new manifest model (a re-skinned water tile from the CC0 Nature pack, `early` wave so boot stays 2.36 MB); play-mode clicks route through `BuildSession` (a pond → `cmd:castLine`, else the existing income-collect). **Verified** by `scripts/verify-fishing.mts` (10 checks: pond loads + places, click→cast routing, nibble, catch +1, popup, journal) + 8 `FishingSystem`/roll unit tests + 2 save tests. **162 tests green**; check + build clean; all prior verifies still pass.

### Added — Cozy polish batch (post-v0.7, in progress)
- **Weather — passing showers + a rainbow (post-1.0 backlog).** The sky occasionally opens into a gentle shower and clears to a low-poly **rainbow** arching over the diorama. A new `WeatherSystem` (world layer): rain is one `LineSegments` draw of ~320 streaks over the current island bounds that eases in, lingers ~15–25 s, then fades; when a daytime shower clears it leaves a **vertex-coloured half-annulus rainbow** (red→violet across the band) that **billboard-yaws to the camera** so it reads from any orbit angle and lingers ~11 s. Rare and slow by construction (a shower every 1–3 min, never busy), self-wires to `chunk:unlocked` so the curtain always covers growth and the arch stays centred, zero per-frame allocation, and **reduced-motion holds the sky calm** (no rain, no fades). Adds ≤2 draws only while a shower is on screen. Verified in `scripts/verify-cozy.mts` (rain + rainbow both draw, no errors) with a rain-and-rainbow beauty shot.
- **Prominent income bubbles — "how much is collectible" at a glance.** The ripen bubble over an income building now surfaces a **readable cream pill showing `● amount`** — but only once the building is *worth a look* (≥55% ripe **and** ≥12 Pops banked), so a busy island stays calm. Below that threshold it's the same subtle fill-dot as before; a fully-ripe prominent pill gently pulses/glows to invite collection. `WorldFx` keeps deriving bubbles from economy state each frame (so it's correct on load / after moves), with the amount text cached by rounded value to avoid per-frame DOM churn. Threshold-gating verified headlessly (full building → pill with the amount; a barely-filled one stays a quiet dot).
- **Background music — drop in your own soundtrack.** Drop a **`public/bgm.mp3`** and it loops automatically (starting on the first user gesture, per browser autoplay policy). Entirely optional — with no file present the new `MusicSystem` is a silent no-op and never errors. A **Music** volume slider in Settings (separate from the **Sound** SFX slider) tunes it live and **persists in the save** (`SaveSettings.musicVolume`, back-filled on older saves with no version bump — round-trip tested). Documented in the README.

### Added — v0.6 "Living Canvas" (feature-complete — awaiting milestone review)
- **Themed / phased lazy-loading (S4):** assets now stream in **waves** instead of one boot download. A pure `content/assetPhases.ts` classifies every model — **boot** (Tiers 1–4 + Islander/Pal agents + world fixtures + auto-tile variants), **early** (Tiers 5–14 with no biome), or **themed:&lt;biome&gt;** (the Sandbar/Spooky/Snowcap Tier 11–13 sets, tagged via a new `ItemDef.theme`) — and the pipeline stamps `phase` into `manifest.json` and **hard-fails if the boot wave tops 3 MB**. First paint now waits on **2.36 MB** instead of loading all **8.58 MB** up front. `AssetRegistry` gained `loadPhase`/`ensure`/`phaseOf` + a pending-promise cache (de-dupes concurrent loads — closes the SYSTEMS S4 double-load-race gap). Later waves trigger in the background: **idle-after-boot → early**, **level-up → the unlocked biome's set** (+ an early safety net), **chunk arrival → the arriving biome's set**. Returning saves **pre-await exactly the phases their placements use** before the world is projected, so every synchronous `show()`/`promote()` keeps its throw-free contract; a still-loading model defers to a pending-show queue (and `makeGhost` falls back to a footprint box) so interactive play is never a crash. A meadow-only island therefore **never downloads** the biome sets. Verified headlessly (`verify-lazyload.mts`, 13 checks incl. a returning early-placement save + adjacent river tiles resolving to variant GLBs, all error-free).
- **UI scale + colour-blind cues (S23, GDD §11):** a **UI size** setting (Small / Normal / Large / Extra large) drives one `--ui-scale` CSS variable; each chrome widget scales around its own pinned anchor edge, so nothing drifts off-screen and the world-anchored layers stay pixel-locked. Colour-blind support lands the GDD §11 line verbatim — the placement **ghost carries a ✓/✕ badge** that swaps on valid↔invalid (icon, not tint alone), the build-hint pill **spells out the block reason** ("✕ That's open sky!"), and unaffordable build cards / survey chips show a coral **✕ afford flag**. Save back-fills `uiScale` with no version bump.
- **Adversarial review pass:** a 6-dimension parallel review (correctness · renderer · async/loading · save/content · perf · a11y) with per-finding verification landed **three confirmed fixes** — (1) a real crash where an auto-tile item's variant GLB could be unresolved while the base was cached mid-lazy-load (the guard now checks the whole kit + defers-load all of it); (2) `deferShow` now catches load failures (no stranded-invisible placement / unhandled rejection) and `AssetRegistry` evicts rejected promises so `ensure` can retry; (3) `makeGhost` now disposes its ghost-owned badge material + placeholder geometry on teardown.
- **River/path auto-tiling (S10):** path and river tiles now **connect** — lay a run and it resolves into straight/corner/tee/cross/end pieces instead of isolated squares. A pure `resolveTileShape(mask)` (the standard 16-mask blob-autotile reduction, unit-tested for all 16 neighbour masks) + per-kit GLB variant data (`content/tileKits.ts`); `PropRenderer` reads a tile's 4 same-kit neighbours, swaps the instanced pool's variant, and ripples the update to neighbours on place/remove/move. Uses the existing Nature Assets 2 connectivity kit (12 new tiny tiles, CC0). **Placements stay the truth — no sim/save change.** Shape resolution verified end-to-end (a tee junction, three ends, straight channels all resolve correctly); the GLB base-orientation calibration is a real-hardware eyeball item. (`stone-path` has only corner/end art, so it degrades gracefully to its plain tile for other shapes.)
- **Edge-anchor placement (S8):** docks, jetties, and moored boats now **must straddle the island's edge** — their footprint needs ≥1 on-island anchor cell and ≥1 off-island cell, so they hang out over the cloud sea. A new `ItemDef.edgeAnchor` flag + a pure `edgeAnchorOrigin(anchor, fp, rot)` grid helper that shifts the origin back for -X/-Z rotations, so a dock reaches **all four** coastlines (not just north/east) as you rotate it. `canPlace` gains a `needs-edge` reason; `BuildSession` routes hovers/clicks through the origin shift. Occupancy is tracked on the off-island overhang too, so two docks can't share it — and it stays claimed even if you later buy the chunk underneath. `decor.dock`/`decor.jetty`/`decor.moored-sloop` are edge-anchored.
- **Tiers 11–14 content (62 items):** the mid/late catalog fills out across four themed sets — **Harbor/Coastal** (Tier 11: leaning/coconut palms, coastal boulders, rowboat, jetty, raft, treasure chest, cannon, boathouse, beached shipwreck…), **Spooky** (Tier 12: five headstone shapes, obelisk, coffin, jack-o'-lantern, brazier, iron fence/gate, dead pines, mausoleum, keeper's crypt…), **Winter/Snowcap** (Tier 13: snow pines, spruce, ice rocks, frozen pond, firewood, camp tent, log cabin, alpine chalet, warm lodge, cocoa stall…), and **Grand Town** (Tier 14: clock tower, castle, temple, grand fountain, monument, town hall, grand market, grain mill, guildhall, manor, townhouse…). `ItemDef.tier` widened to 14 (Tier N unlocks at Level N; gating + "New!" reveals extend automatically). All 62 shipped via the manifest pipeline (Kenney/Quaternius CC0), **every scale computed from the pipeline AABBs**. **Research fanned out across four parallel content-scout agents** that inspected the raw packs and returned verified paths + sizes.
- **Settings & accessibility (S23):** a **frame-rate cap** (Off / 30 / 60 fps) in the loop for battery/thermals, plus the **Time of day** freeze from v0.5 — both persisted. Reduced-motion continues to gate every juice preset + ambient event. (UI-scale and colour-blind cue passes shipped too — see their own entries above.)
- **Island Album (S21):** a cozy scrapbook (📖 button / **J**) that surfaces the lifetime **milestone counters** with tier progress, the **Neighbours** who've moved in, the **Pals** who visit, and a **biome tally** of your districts. Read-only, pulls a fresh snapshot on open, all text via the string table.

The capture renders one fresh frame and reads the canvas back, so it works without a preserved drawing buffer — the marketing shot takes itself.

A deterministic `themeFor(seed, chunk)` rolls the biome (≈58% meadow, the rest sprinkled) so the world is reproducible and a **Survey chip previews the biome** (🌱🏖️🌙❄️) before you buy. ### Added — v0.5 "Little Neighbours" (feature-complete — awaiting milestone review)
- **Ambient life (S19/S20):** the diorama breathes on its own now — **fireflies** twinkle low over the island at night, the odd **shooting star** streaks the dark sky (a soft wish chime + sky sparkle, `event:shootingStar`), and a stray **hot-air balloon** drifts past by day. All driven by the day-night `nightFactor` on long random timers (never busy), reduced-motion-safe.
- **Ambient audio (S22):** a single, well-spaced **birdsong chirp** by day / **cricket trill** by night — the "zero machine-gun artifacts" DoD by construction (one rate-limited voice, 4–10 s apart). Plus a happy **pet chirp** and the tap-to-greet **babble**.
- **Liveliness dividend (S13):** a lively island quietly pays a little extra — every 10 s each resident Islander & Pal chips in a couple of Pops (capped at 18 souls, time-gated so it's never farmable). `LivelinessSystem` is three.js-free + unit-tested.
- **Islanders (S16) — the milestone identity:** little neighbours now **move in as your island gains homes** and **wander it** while you build. Population = **min(Σ house capacity, 12)** and is **monotonic** — removing a home never evicts anyone (no-fail-state covenant). A 16-name roster (`src/content/roster.ts`, move-in order Mo → Pia → Bram → Lulu → …) maps onto the 12 lit, skinned Pack-2 characters; names route through the string table (i18n-ready).
- **Sim/render seam held (Tier C):** `IslanderSystem` (sim, **three.js-free**) owns pure kinematics — plain `{x,z}` positions, headings, per-agent decision timers — and a straight-line "stroll to a nearby walkable cell, re-decide on arrival/obstruction" wander (no A* needed on a small open grid; decisions are seconds apart, only the position integration is per-frame). It exposes an `agents` snapshot the renderer **pulls** each frame (like WorldFx reads economy). New `IslandModel.walkable(wx,wz)` — on-island, not landmark-blocked, free of a solid prop (paths are strollable) — keeps Islanders off benches, huts, and the void.
- **`AgentRenderer` (world):** projects the snapshot into animated meshes — one `AnimationMixer` per agent, an eased **idle↔walk crossfade** driven by `moving`, and **anti-sync** (±10% mixer timescale + random clip phase) so a crowd of the same model never marches in lockstep. Each model is **auto-seated on the ground** via its AABB (no per-model offset guessing). Clones share geometry/materials with the cache.
- **`AssetRegistry` now keeps animation clips** (previously dropped on load) and adds a **skinned-safe clone path** (`cloneAnimated` via `SkeletonUtils.clone`) so each agent's skeleton rebinds independently — a plain `scene.clone()` would share one skeleton and deform every instance in lockstep. Pipeline gained a **required-clip gate** (`npc.*`/`pal.*` must ship `idle`+`walk`) so a bad asset swap fails loudly, not as a silent T-pose.
- **Save v4:** the Islander roster slice (`{ residents }`) + a v3→v4 migration (the sim re-derives residents from home capacity on first start); `normalize` deep-defaults it. Live positions aren't persisted — Islanders simply re-scatter on load.
- **Move-in welcome:** a gentle sparkle + chime where a neighbour appears (`npc:arrived`); the full walk-in-from-the-edge move-in *moment* lands in a later v0.5 slice.
- **Tap-to-greet (user feedback):** click an Islander and they **pause, say a cute one-liner in a springy speech bubble, play a wave/nod emote, and babble** a little WebAudio blip. 14 cozy lines (`content/chatter.ts`, string-table + i18n-ready), a random emote from `emote-yes`/`interact-*`, a one-shot clip layered over the idle/walk blend, and a world-anchored `SpeechLayer` bubble that tracks its speaker. Picking is a raycast against agent meshes, gated so it never fires mid-build. `cmd:clickNpc` → `npc:spoke`/`agent:playClip`.
- **Fixed (user feedback):** Islanders **walked backwards** (facing forward, moving the other way) — the model-yaw offset was a half-turn off; corrected so they face their travel direction.
- **Move-in moment:** a new resident now **enters from the island's cloud edge and strolls in toward the heart of the island**, with a named welcome toast ("Mo moved in! 🏡") + a sparkle/coin burst. Only genuine move-ins do this (load re-spawns scatter quietly).
- **Day-night cycle (S7):** a slow **dawn → day → dusk → night** loop (4-minute day) keyframes the two lights, the sky dome gradient, the cloud sea + puff clouds, and the fog — the whole frame moves together, not just the island. The sun's *direction* stays fixed (no per-frame shadow refit — S5); only colour, intensity, and the gradient move, which carries the mood cheaply. A **Time of day** setting freezes it (Cycle / Day / Dusk / Night), persisted. `TimeOfDay` publishes a `nightFactor` for glow + night-only events.
- **Lantern glow (S7/S20):** soft additive halos fade in over lanterns, lampposts, campfires, and the lighthouse as dusk falls — the "windows glow at dusk" beat. `GlowLayer` tracks placements via the bus (new lamps light up), one shared radial texture, opacity driven by `nightFactor`. **The night beauty shot lands** (ART §3.2): moonlit island, deep-indigo sky, dark cloud silhouettes, warm lamp pools.
- **Pals (S18):** six animals — cat, dog, bunny, chick, pig, cow — **scamper onto the island as it gets lively** (one Pal drawn per 8 nature items, capped at 6, monotonic). `PalSystem` (sim, three.js-free) mirrors the Islander wander with a quicker gait, the occasional grass **nibble** (`eat` clip), and **petting**: tap a Pal → it plays a happy `gesture-positive`, puffs pink **hearts**, and chirps. Rendered by a second `AgentRenderer` at a smaller target height (the same Tier-C pipeline, now per-model scale-normalized so characters and pets share one renderer). New pet assets shipped via the pipeline (Cube Pets CC0). Save slice extended with a monotonic `pals` roster; events `pal:adopted`/`cmd:clickPal`/`pal:petted`.
- **Verification:** `scripts/verify-v05.mts` — boot → no neighbours without homes → build 3 huts → 3 Islanders move in (roster + a mesh each, Mo first) → they wander (Δ>0.4 blocks) staying on walkable ground → roster persists across reload + agents re-spawn (10 checks, zero page errors under swiftshader — skinned clone + mixer confirmed). 9 `IslanderSystem`/`walkable` unit tests (population, Σ-houses capacity, monotonic no-evict, 12-cap, re-spawn-no-dup, a 3000-step on-island wander soak). **121 tests green.**
